📘 pkgdrop - Technical Specification v1.0

1. System Architecture

1.1 Overview

pkgdrop follows a modular handler pattern where each package format has a dedicated installation function.

1.2 Component Flow

main()
  ├── Validate input ($1)
  ├── detect_type($file)
  │     ├── Check extension patterns
  │     └── Return handler type
  ├── Route to install_$type()
  │     ├── pacman → sudo pacman -U
  │     ├── tarportable → extract + symlink
  │     ├── debtap → debtap wrapper
  │     └── appimage → move + chmod
  └── Report result

2. Core Functions

2.1 detect_type()

Purpose: Identify package format
Input: File path
Output: Handler name string
Complexity: O(1) pattern matching 
detect_type() {
    local file="$1"
    case "$file" in
        *.pkg.tar.*|*.pkg.tar.*) echo "pacman" ;;
        *.tar.xz) echo "tarportable" ;;
        *.deb) echo "debtap" ;;
        *.AppImage) echo "appimage" ;;
        *) fail "Unknown" ;;
    esac
}

4. Dependencies

Required:
  - bash >= 5.0
  - coreutils (tar, ln, mkdir, chmod)
  - grep, sed, find

Optional:
  - pacman (for .pkg.tar.*)
  - debtap (for .deb)
  - sudo (for system installs)
